# Contributing Guidelines

Thank you for your interest in contributing to this system architecture!

## Code of Conduct
Please be respectful and professional in all interactions.

## Submitting Pull Requests
1. Fork the repository.
2. Create your feature branch (`git checkout -b feature/AmazingFeature`).
3. Ensure your code follows the `.prettierrc` formatting.
4. Commit your changes (`git commit -m 'Add some AmazingFeature'`).
5. Push to the branch (`git push origin feature/AmazingFeature`).
6. Open a Pull Request.

## Local Development
- Ensure Node.js v22 is installed.
- Ensure Docker is running if the project requires a database or cache layer.
